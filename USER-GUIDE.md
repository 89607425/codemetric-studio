## 完整使用流程

### 第一步：运行 CLI 分析

**目的**：让工具扫描你的 Java 项目，计算度量指标，生成结果文件。

#### 1.1 编译项目（首次使用）

```bash
mvn clean package
```

#### 1.2 运行分析命令

```bash
mvn exec:java "-Dexec.args=analyze --path <你的Java项目路径> --out out --format all"
```

**参数说明**：
- `--path`：要分析的 Java 项目路径（必填）
- `--out`：结果输出目录（默认 `out`）
- `--format`：输出格式，`json` / `md` / `all`

#### 1.3 示例：分析内置样例

```bash
mvn exec:java "-Dexec.args=analyze --path bad-uml-sample --out out --format all"
```

#### 1.4 分析完成后

在 `out/` 目录下会生成：
- `metrics.json`：度量结果数据（前端需要这个文件！）
- `metrics-report.md`：Markdown 格式报告

---

### 第二步：打开前端页面

**目的**：可视化查看度量结果。

#### 2.1 启动本地服务器

在项目根目录执行：

```bash
mvn exec:java "-Dexec.args=serve --port 9090"
```

#### 2.2 打开浏览器

访问：**http://127.0.0.1:9090/web/**

#### 2.3 加载数据

前端页面会**自动尝试加载** `out/metrics.json`。

- 如果已运行过 CLI 分析，数据会自动显示
- 如果没有数据，页面会提示你上传文件

---

### 第三步：使用前端功能

#### 方式一：自动加载（推荐）

1. 确保已运行 CLI 分析，`out/metrics.json` 存在
2. 打开前端页面，数据自动加载
3. 点击左侧导航切换不同度量模块

#### 方式二：手动上传

1. 点击右侧面板的 **"点击上传"** 按钮
2. 选择 CLI 生成的 `metrics.json` 文件，直接选择 `.oom` 设计图文件，或点击“上传 Java 项目”选择整个 Java 项目文件夹
3. 数据加载完成后即可查看

> **注意**：现在支持两类上传文件：
> 1. `metrics.json`：适合展示完整代码度量结果
> 2. `.oom`：适合直接读取类图、用例图、顺序图并进入对应度量模块
> 3. Java 项目文件夹：适合在前端直接触发后端分析并自动返回 `metrics.json` 结果

> **前端直传 Java 项目或使用智能分析前，需要先启动 Java 本地服务：**
> `mvn exec:java "-Dexec.args=serve --port 9090"`
>
> 如果要使用硅基流动智能分析，请先执行：
> `export SILICONFLOW_API_KEY="<你的硅基流动 API Key>"`

## 度量模块说明

### 1. 功能点度量

**用途**：估算软件功能规模

**使用方式**：
- **手动录入**：填写事务功能（EI/EO/EQ）和数据功能（ILF/EIF），系统自动判级
- **自动估算**：点击"自动估算(基于metrics)"，根据已有数据启发式计算

**输出指标**：UFP（未调整功能点）、VAF（调整因子）、调整后 FP

---

### 2. 用例图度量

**用途**：基于用例图估算项目规模

**使用方式**：
- 填写参与者数量（简单/一般/复杂）
- 填写用例数量（简单/一般/复杂）
- 填写技术因子和环境因子
- 点击"计算用例点"或"自动估算"

**输出指标**：UAW、UUC、UUCP、TCF、EF、UPC

---

### 3. 面对象度量（核心模块）

**用途**：展示类的 CK 指标和多态性指标

**使用方式**：
1. 加载 `metrics.json` 后自动显示
2. 雷达图展示 Top 12 风险类
3. 点击类名或图例可高亮单个类
4. 点击"导出XML格式"可下载 XML 报告

**输出指标**：
- CK 核心：WMC、DIT、NOC、CBO、RFC、LCOM
- 多态性：NOP、NOM、NOO、POD、OverrideRatio、OverloadRatio
- 扩展：SK、DAC、MOA、CAM、CIS
- 新增：AIF、MIF

---

### 4. 控制流图度量

**用途**：评估方法逻辑复杂度（圈复杂度）

**使用方式**：
- **手动模式**：粘贴代码，系统统计决策点
- **自动模式**：选择类和方法，直接读取已计算的圈复杂度

**输出指标**：圈复杂度 CC、决策点总数

---

### 5. 代码行度量

**用途**：统计代码行数

**使用方式**：
- **手动模式**：粘贴代码文本
- **自动模式**：点击"使用当前项目分析结果"

**输出指标**：总行数、空行、注释行、有效代码行

---

### 6. 智能分析

**用途**：调用硅基流动 API，结合用户输入、当前软件度量指标和类图/顺序图结构，分析设计质量。

**使用方式**：
- 先加载 `metrics.json`、`.oom` 设计图，或上传 Java 项目完成分析
- 切换左侧“智能分析”标签
- 输入关注点，例如职责划分、耦合风险、顺序图交互复杂度
- 点击“开始智能分析”

**输出内容**：总体结论、关键证据、主要风险和可执行改进建议。

## 常见问题

### Q1：前端页面显示"暂无数据"或空白？

**原因**：没有运行 CLI 分析，或 `out/metrics.json` 不存在

**解决**：
1. 先运行 CLI 分析命令
2. 确认 `out/metrics.json` 文件存在
3. 刷新前端页面

---

### Q2：点击"点击上传"应该上传什么文件？

**答案**：可以上传两类文件

**注意**：
- 支持上传 CLI 生成的 `metrics.json`
- 支持上传 `.oom` 类图、用例图、顺序图
- 支持上传整个 Java 项目文件夹，由前端调用本地接口自动分析
- 不支持直接上传 Java 源代码（.java 文件）
- 不支持直接上传项目压缩包

---

### Q3：如何分析我自己的 Java 项目？

```bash
# 将你的项目放在任意位置，然后运行：
mvn exec:java "-Dexec.args=analyze --path /path/to/your/java-project --out out --format all"
```

---

### Q4：控制流模块的"自动读取"下拉为空？

**原因**：没有先加载 `metrics.json`

**解决**：
1. 先点击"开始类图分析"或上传 `metrics.json`
2. 再切换到"控制流图度量"页签

---

### Q5：功能点/用例点自动估算结果不准确？

**说明**：自动估算是基于启发式规则推断的，仅供参考

**建议**：根据实际业务需求人工复核和调整
