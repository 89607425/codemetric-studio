# 网上药店用例图：功能点逐用例详解

本文针对 `object/网上药店用例图.oom`，按当前程序逻辑对每个用例做逐项解释，适合课堂检查/答辩使用。

---

## 1. 程序使用的判定规则

## 1.1 事务功能（EI/EO/EQ）复杂度判定

- EI：
  - `FTR <= 1`：`DER <= 15 -> low`，否则 `avg`
  - `FTR == 2`：`DER <= 4 -> low`，`DER <= 15 -> avg`，否则 `high`
  - `FTR > 2`：`DER <= 4 -> avg`，否则 `high`
- EO/EQ：
  - `FTR <= 1`：`DER <= 19 -> low`，否则 `avg`
  - `FTR <= 3`：`DER <= 5 -> low`，`DER <= 19 -> avg`，否则 `high`
  - `FTR > 3`：`DER <= 5 -> avg`，否则 `high`

对应权重：

- EI：`low=3, avg=4, high=6`
- EO：`low=4, avg=5, high=7`
- EQ：`low=3, avg=4, high=6`

## 1.2 数据功能（ILF/EIF）复杂度判定

- 若 `RET == 1`：`DET <= 50 -> low`，`DET > 50 -> avg`
- 若 `RET <= 5`：`DET <= 19 -> low`，`20~50 -> avg`，`>50 -> high`
- 若 `RET > 5`：`DET <= 19 -> avg`，`>19 -> high`

对应权重：

- ILF：`low=7, avg=10, high=15`
- EIF：`low=5, avg=7, high=10`

## 1.3 单用例贡献计算

每个用例总贡献 = 事务贡献 + 数据贡献（若配置了 DET/RET）。

## 2. 每个用例详细解释

## 2.1 o69 注册

- 注释：`TYPE=EI;FTR=2;DER=8;STORE=ILF;DET=6;RET=2`
- 事务判定：EI 且 `FTR=2, DER=8` -> `EI.avg`（权重4）
- 数据判定：ILF 且 `RET=2, DET=6` -> `ILF.low`（权重7）
- 单用例贡献：`4 + 7 = 11`
- 解释：注册属于外部输入；输入字段较多，事务中等复杂；落库数据分组不多且字段量低，数据复杂度低。

## 2.2 o70 核实医生信息

- 注释：`TYPE=EQ;FTR=2;DER=5;STORE=EIF;DET=5;RET=1`
- 事务判定：EQ 且 `FTR=2, DER=5` -> `EQ.low`（权重3）
- 数据判定：EIF 且 `RET=1, DET=5` -> `EIF.low`（权重5）
- 单用例贡献：`3 + 5 = 8`
- 解释：核实信息本质查询，事务较轻；外部信息结构简单，数据复杂度低。

## 2.3 o71 回复处方有效无效

- 注释：`TYPE=EO;FTR=1;DER=4`
- 事务判定：EO 且 `FTR=1, DER=4` -> `EO.low`（权重4）
- 数据判定：无 DET/RET -> 不计数据项
- 单用例贡献：`4`
- 解释：对外输出判断结果，涉及数据元素少。

## 2.4 o72 接收处方确认请求
